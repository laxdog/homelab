## 2026-03-27 VGA-only renderer pass

This pass isolated the failure further on `batocera-crt1` with a VGA monitor only.

Key findings:

- The stock `openbox -> emulationstation-standalone -> emulationstation --windowed` session still renders a black framebuffer with only the mouse cursor visible.
- A previous direct `/usr/bin/emulationstation` launch on the same `DISPLAY=:0` was also black, so the wrapper is not the primary failing layer.
- A direct launch with `LIBGL_ALWAYS_SOFTWARE=1` failed immediately with:

```text
glx: failed to create drisw screen
X Error of failed request:  BadValue (integer parameter out of range for operation)
  Major opcode of failed request:  151 (GLX)
  Minor opcode of failed request:  3 (X_GLXCreateContext)
```

- `Xorg.0.log` during the live VGA session shows:

```text
(WW) modeset(0): Present-flip: queue flip during flip on CRTC 0 failed: Device or resource busy
```

Interpretation:

- X itself is not the main failure. Earlier evidence already showed simple X clients rendering on VGA.
- The remaining failure domain is EmulationStation's GLX / Mesa / present path on this VGA session.
